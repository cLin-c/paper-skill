# 扩展辅助提示词

本模块存放非论文核心但科研工作中常见的辅助任务：代码调试、Word/LaTeX/PPT 产出、文件整理、网站组织、合同审查、服务器登录模板、GitHub Pages。除非用户明确需要，否则不要在论文写作主流程中加载本文件。

---

## 1. 代码报错修复

```text
这是我的代码，报错为：[完整报错信息]。

请完成：
1. 定位报错发生的文件、函数和行号；
2. 解释报错原因；
3. 给出最小修改方案；
4. 告诉我具体修改哪里、怎么修改；
5. 修改后运行验证，至少覆盖正常输入、边界输入和异常输入；
6. 如果要求生成完整代码，请输出完整可运行版本。

要求：不要无关重构，不要删除我已有功能。
```

## 2. Word 论文标注修改

```text
请在我上传的 Word 论文中直接修改。

标注规则：
1. 修改前原句标蓝；
2. 修改后句子标红；
3. 不要大段大段标红，只标真正修改的句子或短段；
4. 批注人：[修改者名称]；
5. 每处修改都写清楚修改原因；
6. 保留原有参考文献编号、公式、图表编号和排版结构；
7. 生成修改后的完整 Word；
8. 另生成一份详细修改报告，说明修改前是什么、修改后是什么、为什么修改、位于哪里。
```

## 3. LaTeX 校对与可替换代码

```text
请检查这篇 LaTeX 论文的全文错误，包括单词、语法、图表、公式、参考文献、编号和明显排版问题。

要求：
1. 精确告诉我修改哪一行或哪个可搜索片段；
2. 给出修改前 LaTeX 代码；
3. 给出修改后可直接复制替换的 LaTeX 代码；
4. 修改前/后严格对齐；
5. 解释为什么这样修改；
6. 生成一份 Word 修订报告；
7. 不要只给笼统建议。
```

## 4. PPT 修改

```text
请根据我上传的 PPT 和参考材料，重新生成完整 PPT。

目标：[转博答辩/投资汇报/论文答辩/项目路演]

要求：
1. 先提炼核心科学问题或商业主线；
2. 重组页面结构；
3. 每页只表达一个核心观点；
4. 图示、表格、流程图要美观大气、学术化；
5. 保留原有重要数据和成果；
6. 新增必要的研究框架、技术路线、预期成果、实验平台、应用场景；
7. 生成完整 PPT，不要只给文字建议。
```

## 5. 文件整理

```text
请整理路径 [桌面路径] 下的文件。

要求：
1. 先扫描文件，不要立刻移动；
2. 按科研项目、论文、数据、图片、视频、合同、财务、软件安装包、临时文件等分类；
3. 生成整理方案给我确认；如我已授权，则执行整理；
4. 移动后生成路径导览 Word；
5. 导览必须详细到每个文件名和后缀；
6. 桌面不要留下零散文件夹和散落文件；
7. 不得删除文件，除非我明确要求。
```

## 6. 网站更新

```text
请修改网站文件：[HTML路径]。

需求：
1. 新增/更新标签页：[标签页名称]；
2. 根据 [Excel/Word/材料路径] 整理内容；
3. 每个子表格或模块对应一个清晰子模块；
4. 页面美观、清晰、适合团队内部查看；
5. 修改后在本地预览；
6. 确认无明显排版错误后上传服务器并更新网站：[网站地址]。

注意：不要把敏感凭据和个人信息写入公开页面。
```

## 7. 合同与股权协议审查

```text
我是 [角色，例如 CTO/甲方/乙方]。请审查这份股权/代持/期权协议中对我不合理或风险较高的条款。

要求：
1. 逐条指出风险；
2. 说明为什么对我不利；
3. 给出修改建议；
4. 直接生成可替换条款；
5. 对过高违约金、单方解除权、无限连带责任、竞业限制、回购条件、表决权、财税责任、认缴责任等重点审查；
6. 生成修改后的完整 Word；
7. 修改处标红，原条款可标蓝或保留批注。

注意：这不是正式法律意见，最终应由执业律师审核。
```

## 8. 服务器登录指令模板

```text
请根据以下服务器信息生成安全的登录指令和本地 SSH 配置方案。

服务器 IP：[服务器IP]
用户名：[用户名]
认证方式：[密码/SSH Key]

要求：
1. 不要把明文密码写入脚本或公开文件；
2. 优先建议 SSH Key；
3. 给出 Windows PowerShell 下的登录命令；
4. 给出 ~/.ssh/config 配置模板；
5. 如果需要免密登录，说明如何生成公钥、上传公钥、设置权限；
6. 提醒我不要将密钥或密码提交到 GitHub。
```
