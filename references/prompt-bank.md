# 高频提示词模板库

本文件收集 `paper-skill` 的高频实战提示词。使用时将方括号中的内容替换为自己的论文、代码、路径、期刊、材料或任务目标。

使用原则：

- 涉及论文发表、参考文献真实性、法律协议、财务、投资、医疗等高风险任务时，要求联网或读取原始文件逐项核查。
- 涉及账号、服务器、密码、身份证号、手机号、住址、合同主体等敏感信息时，使用占位符，不要把真实信息写入公开仓库或提示词模板。
- 要求生成 Word/PPT/PDF 时，明确输出路径、标注规则、修改者姓名、是否保留原格式、是否生成修改报告。
- 要求代码修改时，明确“先定位错误、再最小修改、最后运行验证”，不要只要求生成完整代码。

## 1. 文献阅读与研究现状

### 单篇文献精读

```text
请你仔细阅读这篇文章，给出：
1. 文章研究的问题；
2. 方法流程；
3. 核心创新点；
4. 实验设置与实验结果；
5. 应用场景；
6. 主要局限；
7. 这项工作是否涉及持续学习，如果涉及，请说明持续学习机制如何实现；如果不涉及，请说明它距离持续学习还缺少什么。

要求：不要泛泛总结，要结合论文中的图表、公式、实验数据和结论逐项分析。
```

### 多篇论文综述撰写

```text
仔细阅读我上传的所有论文，根据这些论文写一篇综述。

主题方向：[研究主题]
引用格式：在正文中用 [论文完整标题] 标注，不要用编号。

输出要求：
1. 先提出一个清晰的综述主题和逻辑主线；
2. 按研究问题、方法类别、关键技术、实验范式、局限与趋势组织；
3. 每一类方法都要说明代表论文、核心方法、优点、局限；
4. 最后总结该领域未来可做的研究方向。
```

### 国内外研究现状调研

```text
请总结当前 [研究领域] 的国内外研究现状。

要求：
1. 分为国外研究、国内研究、工业界进展、学术界进展；
2. 每项工作列出：论文/项目标题、作者或机构、年份、发表期刊/会议、技术路线、实验平台、主要结果、局限；
3. 有公开视频或项目主页的，给出链接；
4. 重点说明不同技术路线之间的区别；
5. 最后给出本领域仍未解决的关键科学问题。
```

### 研究现状表格

```text
请联网检索并整理 [研究方向] 的最新研究现状，生成一个表格，字段包括：
论文标题、发表时间、期刊/会议、期刊/会议级别、方法简述、关键实验、与本工作的区别、是否有代码、是否有视频链接。

要求：
1. 优先选择近 3 年 SCI 一区、IEEE/ACM 顶会、Nature/Science 子刊论文；
2. 所有参考文献必须真实存在；
3. 每条文献都要给出 DOI、官网或论文链接；
4. 不确定的信息必须标注“需进一步核查”，不能编造。
```

## 2. 论文审稿、修改与润色

### SCI 一区全文深度修改

```text
你拥有 SCI 一区期刊长期审稿人和工科顶刊论文润色经验，精通学术英文写作逻辑、专业术语规范、句式精简、逻辑递进、语法错误、学术表达地道化、格式规范、段落衔接、摘要/引言/实验/讨论/结论分模块优化。

现在对我提供的完整论文全文进行深度大修，严格遵守以下要求：

1. 全文双版本对照标注：
   - 原文未修改内容无底色；
   - 待修改原始语句/词汇标注蓝色；
   - 优化替换后的修订内容标注红色；
   - 区分删除、新增、替换三类修改。

2. 每一处修改都配套说明：
   - 修改前原文；
   - 修改后定稿；
   - 修改理由，包括语法错误、学术用词不地道、句式冗长、逻辑断层、中式英语、期刊表达习惯、专业术语误用、语句歧义、段落衔接薄弱、重复冗余等。

3. 修改范围：
   摘要、关键词、引言、相关工作、方法、实验设置、结果分析、讨论、结论、参考文献描述。

4. 禁止大面积改变核心创新、实验数据和核心结论。只优化语言表达、行文逻辑和学术规范。

5. 额外单独列出“一区审稿视角补充建议”，指出逻辑漏洞、论证缺失、实验描述模糊、图表不足、参考文献风险等。

6. 修改者统一标注为：[修改者名称]。

输出：
1. 逐条修改批注；
2. 完整无批注定稿版全文；
3. 如要求生成 Word，请生成完整 Word 文件并保留原文格式。
```

### 审稿人视角找拒稿风险

```text
你是 [目标期刊/会议] 的审稿人和编委，请以严格审稿标准审阅这篇论文。

请检查：
1. 逻辑链是否完整；
2. 核心创新是否像方法/算法，而不是工程系统拼装；
3. 理论、公式、变量定义是否自洽；
4. 实验是否足以支撑主张；
5. 对比实验、消融实验、泛化实验是否缺失；
6. 参考文献引用内容是否匹配；
7. 图表、表注、图注、术语、缩写是否规范；
8. 是否存在会导致直接拒稿的致命问题。

输出：
1. 最终审稿决定倾向；
2. 录用概率区间；
3. 致命问题清单；
4. 不补实验情况下的可执行修改方案；
5. 需要补实验时的实验清单；
6. 逐处告诉我修改哪里、替换成什么句子、为什么这样改。
```

### 初审拒稿后转投一区修改报告

```text
我被初审编辑直接拒稿了。请根据编辑意见和论文全文，帮我重新规划转投 SCI 一区的修改方案。

1. 分析被拒的根本原因：选刊不匹配、创新表达弱、工程系统化、实验不足、写作逻辑弱、参考文献不足等；
2. 推荐适合转投的一区期刊，并说明匹配理由；
3. 将论文从“系统/框架”重构为“方法/算法”的故事线；
4. 提炼新的核心算法名称、中文解释和英文缩写；
5. 重新组织摘要、引言、相关工作、方法、实验、结论的叙事逻辑；
6. 列出必须补充和可选补充的实验；
7. 给出逐章修改清单；
8. 所有英文术语都给出中文翻译。

输出：生成详细具体的修改报告，适合直接复制进 Word。
```

```text
3. 参考文献编号必须按正文出现顺序排列；
```

```text

要求：

2. 分主题组织，而不是简单罗列论文；
3. 评价性表述必须有参考文献支撑；
```

### 方法部分重写与公式审查

```text
你是 SCI 一区/TRO/TIE/RCIM 审稿人，请基于我当前论文全文，对方法部分进行重写和审查。

重点检查：
1. 方法是否具有算法创新性，是否避免写成工程系统拼装；
2. 数学原理、公式推导、变量定义是否自洽；
3. 变量是否符合论文规范：变量斜体、函数/常量正体；
4. 是否存在英文单词式下标，如 p_cmd，应改为简洁数学符号；
5. 伪代码是否符合标准算法格式；
6. 方法叙事是否围绕核心创新逐层展开。

输出：
1. 修改后的完整方法部分；
2. 公式以可渲染 LaTeX 形式给出；
3. 修改前/修改后/修改原因逐项说明；
4. 如有公式或变量错误，明确指出如何改。
```

### 实验部分重写

```text
你是 SCI 一区审稿人，请结合当前论文全文和实验结果，对实验部分进行重写。

要求：
1. 实验部分必须围绕核心创新逐层验证；
2. 按“实验目的 -> 实验设置 -> 评价指标 -> 结果 -> 讨论”组织；
3. 删除冗余和重复叙述；
4. 补充必要的基线对比、消融、泛化、鲁棒性和失败案例分析；
5. 对每个补充实验说明为什么要做、怎么做、测什么指标、如何证明创新；
6. 表格和图必须服务于论证，不要为了列数据而列数据；
7. 输出可直接放入论文的完整实验部分。
```

## 3. 参考文献与引用真实性

### 全文参考文献核查

```text
请详细检查全文参考文献以及对应引用内容是否正确。

检查目标：
1. 参考文献是否真实存在；
2. 题名、作者、年份、期刊/会议、DOI 是否正确；
3. 正文引用编号是否按出现顺序排列；
4. 是否存在中间漏引、重复引用、编号错乱；
5. 论文中对该文献的描述是否与文献真实内容匹配；
6. 表格中引用的数据是否真实来自对应文献；
7. 不确定的信息必须标注“需要人工复核”，不要编造。

输出：
1. 参考文献问题清单；
2. 正文引用位置修改建议；
3. 错误引用的替换文献建议；
4. 若生成 Word，请在文中直接标注：原句蓝色，修改句红色，并生成详细修改报告。
```

### 表格数据对应文献核查

```text
请重点检查表 [表编号/表名] 涉及的参考文献和数据。

要求：
1. 逐项核查表格中的成功率、实验次数、测试集物体数量、数据集名称、硬件平台等；
2. 判断“未报告”是否真实未报告，还是引用错了；
3. 若数据不准确，请给出正确数据、来源位置和修改建议；
4. 如果文献没有明确报告，不要自行推断为准确值；
5. 输出一个可直接替换表格的修订版。
```

## 7. 实验方案与研究故事线

### 核心创新重构与标准化实验方案

```text
你是 SCI 一区工科 AI 领域审稿专家。结合我上传的论文原稿与专属研究故事线，完全重构、替换论文全部核心创新点，以故事线为唯一研究主线，撰写一份可直接复制到 Word 的标准化实验方案。

内容必须包括：
1. 重构后的完整论文核心创新概述；
2. 实验整体论证逻辑；
3. 标准化实验环境：硬件、系统、语言、框架、依赖库、数据划分、训练超参；
4. 四大类实验清单：基线对比、模块消融、超参敏感性、复杂场景泛化；
5. 每项实验写明：实验目的、操作流程、实验组/对照组、控制变量、为什么必须做、对应验证哪条创新；
6. 完整评价指标体系：数学定义、数值含义、选用理由、对应创新；
```
