# Quality Gates & Manuscript Assessment Module

> **Why gates matter:** Research (lishix520/academic-paper-skills) shows that manuscripts passing a structured 7-dimension quality gate before submission have significantly higher acceptance rates. The threshold (≥28/35) filters manuscripts that need targeted improvement from those ready for submission.

## 1. 7维度35分质量评估

在进入下一写作阶段或提交前，对论文进行7个维度的系统评分。**≥28/35 方可继续，<28 需定向改进。**

### 完整质量评估
```
你是顶级期刊匿名审稿人，对这篇论文进行7维度35分质量评估：

【维度1】原创性（Originality）— 满分5分
评估标准：
5分：完全新颖的方法/发现，对领域有实质性推进
4分：有创新但建立在现有工作的扩展上，贡献明确
3分：有一定新意，但创新点较小或局限于特定场景
2分：主要是已有方法的组合/应用，创新有限
1分：与已发表工作高度重叠，无明显新贡献
我的评分理由：[AI填写]

【维度2】论证质量（Argumentation）— 满分5分
评估标准：
5分：每个核心论点都有充分的实验/理论支撑，逻辑严密
4分：主要论点支撑充分，个别细节可加强
3分：核心论点支撑基本到位，但存在1-2个逻辑跳跃
2分：多处声明缺乏直接证据，靠间接推断
1分：核心论点与提供的证据脱节
我的评分理由：[AI填写]

【维度3】文献覆盖（Literature Coverage）— 满分5分
评估标准：
5分：全面覆盖相关工作，包括最新进展（2024-2026），无明显遗漏
4分：覆盖主要相关工作，少量最新进展可补充
3分：覆盖基本工作，但遗漏了1-2个重要方向或近期关键论文
2分：文献明显不足，存在重要的相关工作未被引用
1分：文献综述浅显，大量相关工作缺失
我的评分理由：[AI填写]

【维度4】方法严谨性（Methodological Rigor）— 满分5分
评估标准：
5分：实验设计严谨，控制变量完整，统计分析恰当，可重复
4分：实验设计合理，有少量可加强的控制实验
3分：主要实验到位，但缺少消融实验或对比基线不够全面
2分：实验设计存在明显缺陷，结论无法支撑
1分：方法描述不完整，实验无法重复
我的评分理由：[AI填写]

【维度5】语言清晰度（Clarity）— 满分5分
评估标准：
5分：表达精确，逻辑流畅，术语使用一致，图表自解释
4分：整体清晰，有少量表达不够精确的地方
3分：主要意思清楚，但有几处歧义或复杂句影响理解
2分：多处表达模糊，读者需要猜测作者意图
1分：大量语言问题影响论文可读性
我的评分理由：[AI填写]

【维度6】学术影响（Academic Impact）— 满分5分
评估标准：
5分：解决了领域核心问题，预计被广泛引用，有重大应用价值
4分：对特定方向有明显推进，有实际应用价值
3分：在特定场景有价值，影响范围有限
2分：理论或实践价值有限，受益面较窄
1分：难以看出对领域的实质推进
我的评分理由：[AI填写]

【维度7】技术准确性（Technical Accuracy）— 满分5分
评估标准：
5分：所有公式正确，数据一致，图表无误，引用真实
4分：技术内容基本正确，有少量需要核实的细节
3分：主要技术内容正确，有1-2处值得商榷的表述
2分：存在技术错误或数据前后不一致
1分：多处明显技术错误影响论文可信度
我的评分理由：[AI填写]

---
总分：[D1+D2+D3+D4+D5+D6+D7] / 35

判定：
≥ 28分 → ✅ 达标，可进入下一阶段/提交
24-27分 → ⚠️ 需改进，重点提升得分最低的2个维度
< 24分 → ❌ 需重大修改，给出优先级改进路线图

对每个得分≤3的维度，给出具体的改进建议和示例
```

### 快速5分钟评估（快速过滤）
```
对这篇论文做快速质量评估（各维度一句话判断）：

原创性：[高/中/低]——[一句理由]
论证质量：[强/中/弱]——[一句理由]
文献覆盖：[全面/基本/不足]——[一句理由]

语言清晰：[清晰/一般/模糊]——[一句理由]

技术准确：[无误/少量问题/有错误]——[一句理由]

预估分数：[X]/35
最需要改进的1个维度：[xxx]
是否建议进入提交阶段：[是/否/需改进后再评估]
```

## 2. 分阶段写作门控

在写作的关键节点设置质量门控，防止带着根本缺陷进入后续阶段。

### 写作前门控（研究计划评估）
```
在开始写作前，评估我的研究计划是否满足写作条件：

我的研究计划：
- 研究问题：[问题]
- 方法：[方法]
- 已完成实验：[描述]
- 主要结果：[结果]
- 目标期刊：[期刊]

写作前门控检查：
□ 研究问题是否已通过FINER评分（≥18/25）？
□ 核心实验是否已完成（而非计划中）？
□ 主要结论是否在统计上显著（若适用）？
□ 是否有明确的消融实验支持设计选择？
□ 是否与至少3个合理的基线方法做了对比？
□ 是否确定了目标期刊并了解其格式要求？
□ 所有实验是否可重复（代码/数据/环境已记录）？

未满足的条目：[列表]
建议：[继续写作/先补充xxx再写作]
```

### 引言完成门控
```
我完成了引言的初稿，进行引言质量门控：

引言文本：[粘贴引言]

检查项：
□ 第一段是否在3-5句内完成背景铺垫，直击研究意义？
□ 研究空白（Gap）是否明确、具体，且有文献支撑（≥2条/gap）？
□ 本文贡献是否以列表形式清晰列出（3-5条，避免模糊）？
□ 贡献声明是否与后文实验相匹配（不过度承诺）？
□ 引言是否以"论文组织结构"段落结尾（若期刊要求）？

通过门控 ✅ / 未通过 ❌（需改进：xxx）后进入方法写作
```

### 方法完成门控
```
我完成了方法部分，进行方法质量门控：

方法文本：[粘贴方法部分]

检查项：
□ 方法是否完整可重复（他人能根据描述复现实验）？
□ 所有超参数是否明确给出（值、范围、选择依据）？
□ 数据预处理步骤是否完整描述？
□ 评价指标是否在此定义（而非仅在结果中出现）？
□ 若有创新模块，是否给出了数学公式/伪代码？
□ 是否说明了实现环境（框架、GPU、版本号）？

通过门控后进入结果写作
```

### 结果完成门控
```
我完成了结果部分，进行结果质量门控：

结果文本：[粘贴结果部分]

检查项：
□ 所有结果陈述是否有对应的表格/图表支撑？
□ 主要结果是否与引言贡献声明一一对应？
□ 消融实验是否覆盖所有主要设计选择？
□ 与基线的对比是否使用相同的评价指标和测试集？
□ 统计显著性是否报告（均值±方差，p值，或置信区间）？
□ 图表是否自解释（看caption能理解图的结论）？

通过门控后进入讨论/结论写作
```

## 3. R&R可追溯矩阵

返修（Revise & Resubmit）中，作者常声称进行了修改但实际上对应位置找不到变更。R&R可追溯矩阵系统性核查每条声称的修改。

### 生成R&R可追溯矩阵
```
我的论文已完成R&R修订，请生成可追溯矩阵来验证修改的完整性：

审稿意见摘要：
R1.1：[审稿意见1]
R1.2：[审稿意见2]
R2.1：[审稿意见3]
...

我在回复信中声称的修改：
针对R1.1：在[章节X第Y段]修改了[内容Z]
针对R1.2：新增了[实验/段落/图表]
...

[粘贴修订稿相关段落]

请生成可追溯矩阵：

| 意见编号 | 审稿人意见（摘要） | 声称的修改 | 修改位置 | 位置可找到？ | 修改是否响应意见？ | 状态 |
|---------|----------------|----------|---------|------------|----------------|------|
| R1.1 | [意见] | [声称修改] | [页码/章节] | ✅/❌ | ✅/⚠️/❌ | 通过/问题 |
| R1.2 | [意见] | [声称修改] | [页码/章节] | ✅/❌ | ✅/⚠️/❌ | 通过/问题 |

对标注为⚠️/❌的行，给出：
1. 问题描述（找不到修改 / 修改不响应意见）
2. 修复建议（在哪里补充什么内容）
```

### 回复信与修订稿一致性核查
```
请核查我的回复信与修订稿的一致性：

我声称的修改：
"[回复信中的具体声明，例如：We have added ablation experiments comparing A and B in Section 4.3]"

修订稿对应位置：
[粘贴修订稿Section 4.3]

核查项：
1. 声称添加的内容是否真实存在于对应位置？
2. 添加的内容是否真正响应了审稿意见的核心关切？
3. 是否存在"形式上回应但实质未改"的情况？
4. 回复信的语气是否与修改力度匹配？

核查结果：通过/有问题（问题描述）
若有问题，给出修复方案：修改回复信 or 补充修订稿
```

### 审稿意见遗漏检查
```
在提交修订稿前，检查是否有审稿意见被遗漏：

所有审稿意见：
[粘贴完整审稿意见]

我的回复信：
[粘贴完整回复信]

请核查：
1. 每条审稿意见（包括次要意见和措辞类）是否在回复信中均有回应？
2. 是否有被我标记为"not applicable"但审稿人可能不认同的意见？
3. 是否有意见被回应了但实际未在修订稿中体现？
4. 回复信的编号是否与审稿意见编号一一对应？

遗漏清单：[列出未回应的意见编号]
建议：对每个遗漏意见给出回应模板
```

## 4. Devil's Advocate自审

Devil's Advocate（魔鬼代言人）机制：AI扮演最挑剔的审稿人，至少持续3轮质疑，只有你提出实质证据才会撤回质疑。这是对抗AI和人类共同存在的"自我认同偏差"的最有效工具。

### Devil's Advocate完整自审
```
你现在扮演最挑剔的顶级期刊审稿人（Nature/Science级别），对这篇论文进行Devil's Advocate批判性审查。

规则：
1. 你必须至少提出5个实质性批评，不能是表面意见
2. 除非作者（我）提供了真实的论文内容或数据作为反驳证据，否则你不能撤回质疑
3. 允许撤回质疑的条件：我引用了论文中真实存在的实验/数据/方法描述
4. 不允许撤回质疑的理由："我们认为这不重要" / "这是常见做法" / "其他论文也这样"
5. 每轮结束后，给出"尚未解决的核心质疑"列表

论文摘要/核心内容：
[粘贴你的论文]

开始第1轮批判：
[AI给出5个最严厉的批评]

---
你的反驳（提供论文中的具体证据）：
[你回应]

---
第2轮审查（坚持未解决的质疑，提出新质疑）：
[AI继续]

---
[重复至少3轮]

最终评估：
- 已有充分证据解决的质疑：[列表]
- 仍需改进的核心弱点：[列表]
- 这些弱点是否是致命弱点（导致拒稿）还是可在回复信中处理的一般弱点？
```

### 针对性弱点挖掘
```
我认为这篇论文的潜在弱点在[方法创新度/实验规模/数据集/对比基线]方面。

请扮演专攻这个方向的审稿人，针对性提出最严苛的质疑：

1. 如果审稿人是这个领域的方法竞争者，他最可能质疑什么？
2. 如果审稿人特别关注实验严谨性，他会提出哪些"缺失实验"？
3. 如果审稿人对我们的gap声明持怀疑态度，他会提供哪些反例？
4. 如果期刊编辑考虑"这篇论文适合我们的期刊吗"，什么会让他犹豫？

对每个质疑，给出：
a. 质疑的严重程度（致命/严重/一般）
b. 如果论文中已有应对，告诉我在哪里
c. 如果没有应对，给出解决方案（补充实验/修改论述/调整期刊选择）
```

### 同行先审模拟（Pre-submission Peer Review）
```
在投稿前，请模拟3位不同背景审稿人的评审意见：

审稿人1（方法专家）：专注方法创新性和技术正确性
审稿人2（应用专家）：专注实验设计和实际应用价值
审稿人3（写作质量审查）：专注论文结构、清晰度和逻辑

对每位审稿人，生成：
- 整体评价（Accept / Major Revision / Minor Revision / Reject）
- 主要优点（2-3条）
- 主要批评（3-5条，具体到章节/段落）
- 必须修改的内容（若接收需满足的条件）

Cross-review综合：
- 三位审稿人的共识优点
- 三位审稿人的共识问题（最需解决）
- 编辑最可能关注的决定性因素

[粘贴你的论文或摘要+关键部分]
```

## 5. 提交前最终核查

### 全面提交前清单
```
在点击提交之前，完成最终综合核查：

内容质量核查：
□ 7维度35分评估是否≥28分？（若未做，先做评估）
□ FINER研究问题评分是否≥18/25？
□ 所有引用是否通过citation-integrity.md的核查？
□ 投前期刊适配检查是否通过（journal-strategy.md）？

写作质量核查：
□ 摘要是否≤要求字数，且包含背景/方法/结果/意义四要素？
□ 引言gap声明每条是否有≥2条直接引用？
□ 方法部分是否完整可重复？
□ 所有图表是否有独立的、信息完整的caption？
□ 结论是否与引言的贡献声明相呼应？
□ 去AI化检查是否完成（paper-skill Section 15）？

技术核查：
□ 所有公式是否经过核对（无符号冲突/下标错误）？
□ 图表分辨率是否达到要求（通常300 dpi+）？
□ 所有交叉引用（Figure 1, Table 2, Equation 3）是否正确？
□ 页码、行间距、字体是否符合期刊要求？

文件核查：
□ 主稿是否为匿名版本（若盲审）？
□ 补充材料是否完整上传？
□ Cover Letter是否已准备？
□ 所有作者是否确认并同意投稿？
□ 是否有需要披露的利益冲突（即使是"无冲突"也需声明）？

提交系统核查：
□ 所有必填字段是否已填写（关键词/类别/推荐审稿人）？
□ 文件格式是否符合要求（PDF/Word/LaTeX zip）？
□ 图片文件是否单独上传（若期刊要求）？

全部完成 ✅ → 可以提交
有未完成项 ❌ → 逐一解决后再提交
```
