# Citation Integrity & Anti-Hallucination Module

> **Why this matters:** A 2026 study (Zhao et al.) found 146,932 hallucinated citations in AI-assisted papers published in 2025. This module systematically detects and prevents citation fabrication.

## 1. AI引用幻觉扫描

### 全文引用幻觉扫描
```
你是学术诚信审查员。请对这篇论文进行引用幻觉扫描：

扫描步骤：
1. 提取论文中所有引用（编号或作者-年份格式）
2. 对每条引用逐一核查：
   - 标题是否与参考文献列表一致（逐字对比）
   - 作者姓名是否合理（不含随机字符或明显生成痕迹）
   - 发表年份与期刊卷期号是否匹配
   - DOI格式是否合法（若有）
   - 期刊名称是否为真实期刊（非伪造期刊名）
3. 特别标注：被引用内容与该文献实际结论是否一致

输出格式：
| 引用编号 | 标题 | 风险等级 | 问题描述 |
|---------|------|---------|---------|
| [1] | ... | 🔴高/🟡中/🟢低 | ... |

生成引用核查报告，高风险引用给出替换建议
```

### 摘要与引言引用重点扫描（高风险区域）
```
摘要和引言是AI幻觉引用高发区。请重点核查：

1. 摘要中的所有引用（若有）是否在参考文献列表中存在？
2. 引言第一段的"权威引用"（通常是为了支持研究重要性的宽泛声明）：
   - 引用的数字/数据/结论是否可在原文中找到？
   - 引用是否支持所在句子的具体声明（而非仅仅主题相关）？
3. "According to [X], ..."句式：[X]的原文是否真的说了这句话？

对每条可疑引用，提供：
- 原始引用声明（论文中的句子）
- 应有的原文证据（你能核实的内容）
- 风险评估和处理建议
```

## 2. 单条引用核实

### DOI/标题核实
```
请核实以下引用的真实性：

引用信息：
[作者]. [标题]. [期刊]. [年份]. DOI: [doi]

核实要求：
1. DOI格式验证：是否符合 10.XXXX/YYYY 格式？
2. 期刊验证：[期刊名]是否为真实期刊（Scopus/Web of Science收录）？
3. 标题合理性：标题是否与声称的研究方向一致？
4. 引用声明核查：你用这条引用支持的论点是"[你的论点]"——这条引用能支撑这个论点吗？

如果无法在线核实，给出我应该在哪个数据库（PubMed/IEEE Xplore/ACM DL/Google Scholar）中手动验证的搜索策略
```

### 数据/统计声明溯源
```
我的论文中有如下声明：
"[声明，如：深度学习在X任务上达到Y%准确率（[引用]）]"

请帮我核实：
1. 该引用文献是否真实存在？
2. 该文献是否真的报告了这个具体数字/结论？
3. 上下文是否一致（如：同样的任务定义、相同的评估指标）？
4. 如果数据不一致，建议我：
   a. 修改声明（更准确描述原文结论）
   b. 寻找更合适的引用
   c. 删除该声明（若无可靠支撑）

[粘贴被引文献的摘要或相关段落，供核实]
```

## 3. Trust-Chain 溯源

Trust-Chain 是将**核心论点 → 引用文献 → 原文段落 → 结论匹配**三层锚定的溯源框架，确保每个重要声明都有可追溯的证据链。

### 为核心论点建立Trust-Chain
```
请为我论文的以下核心论点建立Trust-Chain（证据溯源链）：

论点列表：
1. [核心论点 A]
2. [核心论点 B]
3. [核心论点 C]

对每个论点，建立三层锚定：

层级1 — 引用定位：
  → 支持该论点的引用编号和文献完整信息

层级2 — 原文段落：
  → 引用文献中实际支持该论点的原文段落（逐字引用或准确转述）

层级3 — 结论匹配：
  → 我的论点声明 vs 原文结论：是否完全一致？是否存在过度解读？

输出格式：
论点A：[论点]
├── 引用：[文献完整信息]
├── 原文：[原文段落]
└── 匹配度：✅ 完全支持 / ⚠️ 部分支持（差异：xxx）/ ❌ 不支持

对⚠️和❌的论点给出修改建议
```

### 研究空白（Gap）引用溯源
```
我论文引言中声称了以下研究空白：

研究空白：[描述现有方法/知识的局限性]
支撑引用：[引用列表]

请帮我核实这个"gap声明"的引用链：
1. 每条引用是否真的表明了这个局限性？还是仅仅是相关工作？
2. 这个gap是否被声称的引用明确提出，还是我的推断？
3. 是否还有其他更直接支持这个gap的引用我应该找到？

建议每个gap至少有3-5条直接支撑引用，而非仅做背景铺垫。
给出引用补强建议和修改后的引言段落。
```

## 4. R&R修订引用核查

返修（Revise & Resubmit）后，引用常发生增删，需专项核查。

### R&R前后引用变更核查
```
我对论文进行了R&R修订，引用发生了变化。请帮我核查：

修订说明（来自回复信）：
"针对Reviewer 2的意见，我们在引言中增加了对[XXX方法]的讨论，并新增了引用[新引用列表]"

核查要求：
1. 回复信声称新增的引用是否真实出现在修订稿对应位置？
2. 新增引用是否真实存在（非幻觉）？
3. 新增引用是否真的支持声称的论点？
4. 是否有引用被删除而回复信未提及（需确认是否合理）？

[粘贴：修订稿相关段落 + 原稿相关段落 + 回复信对应段落]

输出：核查通过 ✅ / 核查有问题 ⚠️（具体问题描述）
```

### 审稿人质疑引用时的应对
```
审稿人质疑了我的引用：
"[审稿人原话，如：The authors cite [X] to support Y, but [X] does not directly address Z]"

请帮我：
1. 判断审稿人质疑是否成立（根据引用文献内容）
2. 如果成立：
   a. 找到更合适的替代引用
   b. 修改论述以准确反映引用内容
3. 如果不成立：
   a. 帮我起草有理有据的澄清回复
   b. 指出原文中支持我论点的具体段落
4. 生成修订后的相关段落和对应的回复信段落
```

## 5. 引用一致性核查

### 参考文献格式一致性检查
```
请对这篇论文的参考文献列表进行格式一致性检查，目标格式：[IEEE/APA7/Nature/IJCAI/MLA/Chicago/Vancouver]

检查项目：
1. 作者名格式（First Last vs Last, F. vs Last F.）是否统一？
2. 期刊名是否统一缩写（或统一全称）？
3. 卷/期/页码格式是否统一？
4. DOI/URL格式是否统一？
5. 会议论文与期刊论文的格式区分是否正确？
6. 在线优先出版（Online First / Advance Access）的处理是否一致？
7. 中文文献的处理方式是否符合目标期刊要求？

逐条列出不一致项，给出统一后的正确格式，生成完整的规范化参考文献列表
```

### 正文引用与参考文献对应核查
```
请核查论文正文引用与参考文献列表的对应关系：

1. 正文引用但参考文献列表中缺失的引用（漏列）
2. 参考文献列表中有但正文未引用的文献（冗余）
3. 引用编号顺序是否符合首次出现顺序（IEEE/Vancouver风格）？
4. 同一文献是否在不同位置使用了不同编号/格式？
5. 自引比例：自引文献占总引用的比例是多少（建议≤20%）？

[粘贴正文 + 参考文献列表]

输出：对应关系核查报告，标注所有异常项
```

## Anti-Hallucination Checklist（提交前必查）

```
在提交论文前，完成以下引用诚信自查清单：

□ 所有DOI已手动验证（在doi.org或期刊网站可打开）
□ 参考文献列表中无拼写异常的期刊名（如："Proceedings of the Internationl Conference"）
□ 没有引用自己未曾阅读过的文献（转引需注明）
□ 所有数据/统计声明已对照原文核实，不存在数字偏差
□ 引言中的"gap声明"每条有≥2条直接文献支撑
□ 所有"According to [X]"类句子已在原文中找到对应段落
□ 参考文献总数与正文引用数一致（无漏列/冗余）
□ 自引比例≤20%（或符合目标期刊要求）
□ R&R修订新增引用已核实真实性

对每个未完成项，给出具体修复操作。
```
