# Community

paper-skill grows from real academic writing problems. The most useful contributions are concrete paper scenarios, anonymized examples, and failure cases.

## What to Submit

Use GitHub Issues for:

- Example requests: a paper-writing workflow you want documented.
- Feature requests: a new reusable skill behavior.
- Bug reports: unsafe or incorrect behavior, especially invented citations, invented experiments, or over-claimed conclusions.

## Good Example Request

```text
Scenario: Chinese master thesis chapter to IEEE-style Methods section.
Target: robotics / IEEE TRO.
Input: Chinese method description + formulas + one experiment table.
Expected output: English Methods section, variable consistency check, and author-input placeholders.
```

## Contribution Principles

- Do not upload confidential manuscripts or private reviewer comments.
- Remove names, affiliations, unpublished data, and sensitive identifiers from examples.
- Prefer small examples that demonstrate one workflow clearly.
- Mark whether the target is SCI, IEEE, Nature-family, thesis, conference, or rebuttal.

## Maintainer Priorities

The project prioritizes:

1. Chinese-to-English submission workflows.
2. Reviewer response and R&R traceability.
3. Citation integrity and anti-hallucination checks.
4. Quality gates before submission.
5. Copyable examples and lab-friendly workflows.
